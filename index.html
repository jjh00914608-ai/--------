Bug Report Case

발견한 이슈를 팀이 빠르게 이해하고 바로 움직일 수 있도록, 재현 가능성과 전달력이 모두 보이는 구조로 정리했습니다.

KAN-7 · UI Consistency

모달 팝업 내 확인 버튼 UI 크기 및 배치가 화면별로 일관되지 않음

동일한 확인 CTA 버튼이 팝업 유형에 따라 서로 다른 크기와 정렬로 노출되어, 사용자가 서비스 규칙을 다르게 인식할 수 있는 현상입니다.

Reproduction Flow

29CM 웹 서비스에 접속 후 로그인 또는 계정 관련 페이지로 이동합니다.
무신사 통합계정 안내 팝업의 확인 버튼 크기와 배치를 확인합니다.

“상품 옵션을 선택해주세요.” 안내 팝업의 확인 버튼 크기와 배치를 비교합니다.

Growth Roadmap

현재는 Manual QA를 중심으로 기본기를 다지고 있습니다. 앞으로는 API 검증, 자동화, 배포 흐름까지 넓혀서 먼저 살피고 먼저 움직이는 QA로 성장하고 싶습니다.

STEP 01

Manual QA 기본기

요구사항 분석, 테스트케이스 설계, 재현 가능한 버그 리포트 작성 완성도 향상

STEP 02

API Testing

Postman 기반 요청·응답 검증, 상태 코드와 예외 응답 확인, 데이터 정합성 검토

STEP 03

Automation

Playwright로 핵심 사용자 흐름 자동화 및 회귀 테스트 구조 이해

STEP 04

CI/CD Collaboration

Git/GitHub, Jenkins를 활용한 테스트 실행 및 결과 공유 흐름 학습
